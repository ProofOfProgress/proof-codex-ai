STRATEGY 03
The Dakota™ Workflow

A full written SOP. Build a master image script, then a video script, and lock them in.

This is a written workflow, not a video. It gives you two reusable scripts: one that drops any product into a clean lifestyle photo, and one that animates that photo into a smooth UGC clip. You build each script once with Claude, test it, then lock it as your master template for every product.

Part 1 — The Image Script (Nano Banana Pro)

Have these three things open before you start: a product reference image (clearer is better), your image tool (Nano Banana Pro or ChatGPT image), and Claude.

Phase 1 — Have Claude write your Nano Banana script

The script Claude writes is what you paste into Nano Banana Pro. You never write it yourself. Paste this into Claude:

Copy
You are a 180-IQ product scriptwriter, and your goal is to create the image that becomes a high-converting TikTok Shop affiliate video. Write me a reusable image-generation script for Nano Banana Pro. The job: take a product reference image and place that exact product into a realistic, aspirational-but-attainable home, as a vertical 9:16 photorealistic lifestyle photo. It must work for ANY product and defend against every common failure. Make sure it covers: the product's exact shape, parts, and part count; locking the exact color; building a brand-new environment instead of copying the reference background; placing the product where it'd really be used; correct real-world scale; gravity and how the product stands; bundles and multipacks showing every piece; framing that leaves room on both sides for a later camera pan; realistic materials and lighting that looks good but never unrealistic; and a clear list of things to never include (mirrors, fire, moving water, people, other brands). Write it as one continuous set of rules I can paste straight into Nano Banana Pro. Nano Banana Pro has no character limit, so make the script as detailed and complete as it needs to be to get the product perfect. Don't cut anything important, just keep every line a clear rule rather than filler.
The output Claude hands back, not this prompt, is what goes into Nano Banana Pro.

Then ask for three variations. Reply:

Copy
Now give me three different versions of that script. Same goal and rules, but vary the wording, the order, and how strongly each emphasizes product accuracy vs. environment. Label them Version 1, 2, and 3.

Phase 2 — Test all three

Pick one test product. Generate from all three versions using the same product image, 2 to 3 images per version so you are not judging off one lucky or unlucky result. Then score each image against this checklist.

✅
Image Score Checklist
•
Exact product, right number of parts, nothing added or missing
•
Color is a true match to the reference
•
Background is a NEW environment, not the reference’s
•
The setting makes sense for this product
•
Scale is believable (not too big or small for the room)
•
Open space on both sides (room to pan later)
•
Looks real, not CGI or fake-perfect
•
Zero banned items (mirrors, fire, water, people, other brands)

Write down what broke. Note which version scored best and list the specific failures, like "Version 2 changed the color" or "Version 1 added a lid that isn’t on the product."

On backgrounds: the best environment is a simple, realistic space where the buyer can already picture the product as if it were theirs. Sometimes that fits the product category (a blender on a kitchen counter, a dumbbell in a home gym), other times it is just a clean, elevated space. Either way keep it simple, believable, and never busy or cluttered.

Phase 3 — Improve and retest

Keep a running list of every issue. Fix the biggest problem first, one at a time, and retest in batches after each change so you know what actually helped. These are the usual suspects:

🛠️
Most Common Fixes
•
Scale: product sized too big or too small for the room
•
Color: the shade is off, or the lighting shifted it
•
Gravity: product floating, tilting, or standing in a way it physically couldn’t
•
Bundles & multipacks: not every piece showing, or parts hidden behind each other
•
Lighting: either flat and bad, or pushed so hard it looks unrealistic

Take your best version, pick the single biggest problem, and tell Claude:

Copy
Version 2 was best, but its main problem is [it changed the color]. Rewrite Version 2 to fix that one thing and tighten the wording so it can’t happen again.
1
Retest as a batch. Run the fixed script several times, re-score with the same checklist. If the issue is solved, cross it off and move to the next one.
2
Prove it covers all bases. Run it on a few hard products: a pair of shoes, a multi-piece bundle, a very large item, and a very small item. Anything breaks, go back and fix one issue at a time.
3
Lock it. Once it passes consistently, save it as your master image script with a version number and date. This is now your starting template for every product.

Part 2 — The Video Script (Kling)

This script only moves the camera. It does not build environments or change the product, your image already did that. If you catch yourself describing a room here, stop and go back to the image script. Have your finished approved image, Kling (kling.ai), and Claude open.

⚙️
Kling Settings (same every time)

Click "Omni does it all"

Video Source 01

720p (1080p works but isn’t necessary)

5-second clip

Phase 1 — Get your first draft

Paste this into Claude:

Copy
You are a 180-IQ product scriptwriter, and your goal is to create a high-converting TikTok Shop affiliate video. Write me a reusable video-generation script for Kling that animates a finished product image into a 5-second vertical 9:16 UGC-style clip. The ONLY job is camera movement, one smooth lateral pan across the product, like a person calmly walking past it. It must NOT change the product, colors, text, or environment. The image is the source of truth. Make sure it covers: keeping the full product visible every second; clear, smooth, gimbal-like motion with no shake; rules for small products (more travel, start offset) vs. large products (keep centered, let the background sweep past); no zoom, tilt, or rotation; wheels that don't roll; nothing morphing or freezing (including the first and last second); and a list of things to never add (people, reflections, pets, fire, water, screens that are on). Write it as one continuous set of rules I can paste into Kling, and keep the whole thing under 3,500 characters maximum. It just needs to be a clear set of rules for the tool to follow, not a long essay.

Then ask for three variations. Reply:

Copy
Now give me three versions of that script. Same goal, but vary how each one balances keeping the product fully in frame against making the movement obvious. Label them Version 1, 2, and 3.

Phase 2 — Test all three

Generate a clip from each version using the same locked image, then score each clip.

✅
Video Score Checklist
•
The FULL product is visible the entire clip, never cut off
•
There is clear, obvious camera movement, not nearly-still
•
The motion is smooth, no shake, jitter, or speed changes
•
The product stays put, no rolling wheels, no sliding
•
Product, colors, and text stay identical, no morphing
•
The first and last second are still moving, not frozen
•
Zero banned items added (people, reflections, fire, water, on-screens)

Write down the best version plus the specific failures, like "Version 1 froze on the last second" or "Version 3 barely moved."

Phase 3 — Improve and retest

Same loop. Fix the biggest problem first, one at a time, testing in batches. Take your best version and tell Claude:

Copy
Version X was best, but its main problem is [it froze at the end]. Rewrite it to fix that one thing and make the rule stricter so it can’t happen again.
1
Retest as a batch and re-score with the same checklist. Solved, cross it off. Not solved, refine again before moving on.
2
Prove it covers all bases. Test on a small product, a large product, and a wheeled product. Small gets more movement, large stays centered while the background sweeps past, wheels stay still.
3
Lock it. Once it passes consistently, save it as your master video script with a version number and date.
A note on pace: you do not need to test after every single generation. Get each script strong out of the gate, run it for a few days, and let your real generations show you what is actually breaking. A good script is strong from the start and then barely touched, not something you are constantly tinkering with.
